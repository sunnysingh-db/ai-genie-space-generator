# AI-Powered Genie Space Generator

Give it a list of tables, get a fully configured Genie Space with metric views, dimensions, measures, and business-friendly sample questions — all generated by AI.

## How It Works

1. You provide a list of fully qualified table names (`catalog.schema.table`)
2. The framework scans their metadata (columns, types, sample data, date ranges)
3. An LLM generates business context, sample questions, and writes `config.yaml`
4. A second LLM pipeline generates dimensions, measures, and joins
5. Metric views are created in Unity Catalog
6. A Genie Space is created with everything wired up

Tables can come from **different catalogs and schemas** — the framework handles cross-schema references automatically.

## Quick Start

Open the notebook and run these cells in order:

| Cell | What It Does |
| --- | --- |
| Cell 2 | Installs `langchain-databricks` |
| Cell 3 | Restarts Python to pick up the package |
| Cell 4 | Set your `table_list` — list of fully qualified table names |
| Cell 5 | Runs auto-configure: scans tables, calls LLM, writes `config.yaml` |
| Cell 6 | Runs the full framework: metric views + Genie Space |


## What Gets Created

- **config.yaml** — business context, sample questions, table list (auto-generated, editable)
- **Metric views** — Unity Catalog views with measures, dimensions, and joins defined in YAML semantic layer
- **Genie Space** — natural language Q&A interface for business users, pre-loaded with sample questions and table descriptions

## Configuration

`config.yaml` is auto-generated by Cell 5 but you can edit it before running Cell 6.

### Key Fields

| Field | Description |
| --- | --- |
| `table_list` | List of fully qualified table names (required) |
| `catalog` / `schema` | Target schema where metric views are created (auto-derived from first table) |
| `genie_space_name` | Display name for the Genie Space |
| `llm_model` | Primary LLM model endpoint |
| `business_domain` | What the business does (plain language) |
| `data_description` | What data is tracked (plain language) |
| `stakeholders_and_decisions` | Who uses this data and for what |
| `additional_context` | Date ranges, categories, relationships |
| `sample_questions` | Business questions the Genie Space should answer |

## Pipeline Steps (Cell 6)

1. **Scan Metadata** — queries `information_schema` for each table, samples rows, infers relationships
2. **Generate Metrics Config** — LLM generates dimensions, measures, joins, sample questions, and business instructions
3. **Create Metric Views** — creates `CREATE VIEW ... WITH METRICS` statements in the target schema
4. **Create Genie Space** — calls Databricks API to create the space with all tables, metric views, joins, and sample questions

## Requirements

- Databricks workspace with Unity Catalog enabled
- Access to Foundation Model APIs (Claude Sonnet/Opus or similar)
- `langchain-databricks` package
- Read access to the tables in your `table_list`
- Write access to the target schema (for metric view creation)

## Troubleshooting

| Problem | Cause | Fix |
| --- | --- | --- |
| `ModuleNotFoundError: langchain_databricks` | Package not installed | Run Cell 2 (install), then Cell 3 (restart Python) |
| `Table or view not found` | Table name misspelled or no access | Check `table_list` entries are fully qualified and you have read access |
| `UNRESOLVED_COLUMN` warnings during metric view creation | LLM hallucinated a column name | Framework auto-recovers — removes the bad column and retries. No action needed |
| Metric view creation fails entirely | Write access missing on target schema | Ensure you have `CREATE VIEW` permission on the target `catalog.schema` |
| `config.yaml` not found | Cell 5 was skipped | Run Cell 5 first to generate the config |
| LLM call times out or fails | Foundation Model endpoint issue | Framework retries across model pool automatically. If all fail, check endpoint status |
| Genie Space shows fewer tables than expected | 30-item API limit | Metric views are prioritised. Reduce table count or review which tables are most needed |
| `Rate limit exceeded` | Too many concurrent LLM calls | Framework handles this with sequential phases and retry logic. Re-run if needed |
| Special characters in catalog/schema/table names | Identifiers with hyphens, spaces, etc. | Handled automatically — all identifiers are backtick-quoted |

## Notes

- The framework uses a model pool (`databricks-claude-sonnet-4-6`, `databricks-claude-opus-4-6`, `databricks-claude-sonnet-4-5`) with automatic failover
- Metric views are created with `CREATE OR REPLACE VIEW`, so re-running is safe
- The Genie Space has a 30-item limit — if you have many tables, metric views are prioritised over raw tables
- Invalid columns generated by the LLM are automatically detected and removed on retry
- Identifiers with special characters (hyphens, spaces) are automatically backtick-quoted
